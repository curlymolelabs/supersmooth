🛫 Pre-flight Checklist

  • E2E test on real Antigravity install
  • Version in package.json is correct (currently 0.1.0)
1
Create an Eclipse Account
Manual

An Eclipse Foundation account is required to sign the Publisher Agreement, which is a prerequisite for publishing to Open VSX.

  • →
    Go to accounts.eclipse.org/user/register
  • →
    Register with your email
  • !
    Fill in the GitHub Username field. It must match the GitHub account you use to log into open-vsx.org.
2
Sign the Publisher Agreement
Manual
  • →
    Go to open-vsx.org, click account icon (top-right), log in with GitHub
  • →
    Click your avatar, go to Settings > Profile
  • →
    Click Log in with Eclipse, authorize the app
  • →
    Click Show Publisher Agreement, read it, click Agree
⚠ The "Eclipse Contributor Agreement" (ECA) is a different thing. You need the Publisher Agreement specifically.
3
Generate an Access Token
Manual
  • →
    On open-vsx.org, go to Settings > Access Tokens
  • →
    Click Generate New Token, enter description (e.g. "supersmooth-publish")
  • →
    Click Generate Token
⛔ Copy the token immediately. It will not be shown again. Store it securely. If lost, delete it and generate a new one.
4
Create the Namespace
Terminal

Your package.json has "publisher": "curly-mole-labs". Create that namespace in the registry:

❯ npx ovsx create-namespace curly-mole-labs -p YOUR_TOKEN
ℹ Creating a namespace does not automatically verify you as owner. To get a verified badge, follow the namespace ownership claim process after publishing.
5
Package the Extension
Terminal

Build the .vsix file from the supersmooth project root:

❯ npx @vscode/vsce package

# Creates: supersmooth-0.1.0.vsix
6
Publish to Open VSX
Terminal

Option A: Publish the .vsix file (recommended for first time)

❯ npx ovsx publish supersmooth-0.1.0.vsix -p YOUR_TOKEN

Option B: Publish from source (packages + publishes in one step)

❯ npx ovsx publish -p YOUR_TOKEN
ℹ Open VSX runs automatic scans for leaked secrets, blocklisted files, and typosquatting. If your extension is rejected, the error will explain which check failed.
7
Verify the Listing
Manual
  • →
    Check open-vsx.org for your extension listing
  • →
    Verify name, description, version, and README render correctly
  • →
    Test installing from a fresh editor: Extensions > search "supersmooth"
✓ If everything looks good, congratulations! Supersmooth is live on Open VSX.
↻
Updating (Future Releases)

For subsequent releases, only Steps 5 and 6 are needed:

# 1. Bump version in package.json
# 2. Run tests
❯ npm test

# 3. Package
❯ npx @vscode/vsce package

# 4. Publish
❯ npx ovsx publish supersmooth-X.Y.Z.vsix -p YOUR_TOKEN
?
Troubleshooting
Error Fix
"You must sign a Publisher Agreement" Go to Settings > Profile on open-vsx.org, click "Log in with Eclipse", then sign the agreement
"Namespace not found" Run npx ovsx create-namespace curly-mole-labs -p TOKEN first
Extension rejected by scan Check for accidental secrets in source. Add // secret-detector:ignore for false positives
Token not working Tokens cannot be recovered after creation. Delete the old one and generate a new token
